Визуальные блоки (Конструктор)

Создание визуальных блоков для конструктора без знания API Blockly

В Spraute Engine встроен мощный Конструктор блоков. Он позволяет пользователям без знаний API Blockly создавать красивые динамические визуальные блоки для своих плагинов. Конструктор читает и генерирует специальный синтаксис на основе комментариев #\.

Единый синтаксис — вам больше не нужно писать отдельный сложный код для генерации скриптов и их обратного парсинга. Движок использует единый синтаксис, который автоматически строит и визуальный интерфейс, и генератор кода, и парсер!

Логика и дизайн блока (Тело)

В конструкторе визуальных блоков (в студии) вы задаёте ID, Категорию, Цвет и Форму через удобный интерфейс. А вот в большом поле "Синтаксис и логика блока" вы описываете то, как блок выглядит и в какой код он превращается.

Основные элементы: row: и input:

Каждая строка визуального блока должна начинаться со слова row:. Дальше идут элементы интерфейса через пробел:

"Текст"

Обычный текст, оборачивается в двойные кавычки

(переменная: тип: значение)

Поле ввода. Типы: number, text, value

[переменная: dropdown(...)]

Выпадающий список с вариантами

dropdown_npc

Системный список NPC (заполняется автоматически)

input: переменная (type: value)

Слот для вложенных блоков (выемка)

Пример базового блока

#\ row: [npc: dropdown_npc] "сказать" (msg: text: "Привет!")
#\ template: {npc}.say("{msg}")

Пример с выпадающим списком

#\ row: [npc: dropdown_npc] "Сказать" (msg: text: "Привет!") "игроку" [player: dropdown(всем: all, случайному: random, ближайшему: nearest)]
#\ if player == "all":
#\   template: for (p in players()) {{ {npc}.say(p, "{msg}") }}
#\ if player == "random":
#\   template: {npc}.say(randomPlayer(), "{msg}")
#\ if player == "nearest":
#\   template: {npc}.say(getNearestPlayer(), "{msg}")

Генерация и парсинг — template:

Чтобы блок генерировал код (и умел парсить его обратно!), просто используйте ключевое слово template:.

#\ row: [npc: dropdown_npc] "сказать" (msg: text: "Привет!")
#\ template: {npc}.say("{msg}")

Как это работает?

  1. Генерация: Когда пользователь ставит этот блок, движок берёт значение из поля npc и msg, подставляет их в шаблон и генерирует код: my_npc.say("Привет!").
  2. Парсинг: Если пользователь напишет в коде my_npc.say("Привет!") и переключится в визуальный режим, движок автоматически по этому же шаблону поймет, что это ваш блок, и сам заполнит поля npc и msg!

Динамические блоки и условия if

Это самая мощная часть конструктора. Она позволяет блоку менять форму (прятать или показывать поля) и менять генерируемый код в зависимости от выбранных опций.

Важно: Используйте Python-подобный синтаксис if (с двоеточием на конце). Отступы (пробелы) имеют значение! Всё, что сдвинуто вправо после if, будет показано/сгенерировано только при выполнении условия.

Комплексный пример — блок "Смотреть"

#\ block: look_at
#\ category: Движения
#\ color: #38bdf8
#\ shape: statement
#\ 
#\ row: [npc: dropdown_npc] "Смотреть" [mode: dropdown(один раз: lookat, всегда: alwayslookat, перестать: stoplookat)] "на" [target_type: dropdown(НИПа: npc, игрока: player, моба: mob)]
#\ 
#\ if mode == "stoplookat":
#\   template: {npc}.stoplookat()
#\ 
#\ if mode == "lookat" or mode == "alwayslookat":
#\   if target_type == "npc":
#\     row: "по имени" [target_npc: dropdown_npc]
#\     template: {npc}.{mode}("{target_npc}")
#\   
#\   if target_type == "player":
#\     input: target_player (type: value) "переменная игрока"
#\     template: {npc}.{mode}({target_player})
#\   
#\   if target_type == "mob":
#\     row: "моб" (target_mob: text: "zombie")
#\     template: {npc}.{mode}("{target_mob}")

Разбор примера:

  • Блок всегда показывает выбор НИПа (npc), выпадающий список mode и выбор цели target_type.
  • Если выбран режим stoplookat (перестать), блок прячет все остальные поля и генерирует код {npc}.stoplookat(). Парсер также поймет этот шаблон!
  • Если выбран режим lookat или alwayslookat (один раз или всегда), блок смотрит на выбранный target_type:
    • Если цель npc, он динамически добавляет строку с выбором имени НИПа и меняет шаблон генерации/парсинга!
    • Если цель player, он добавляет слот для переменной.
    • Если цель mob, добавляет поле ввода для ID моба.

Генератор кода и парсер

Вам больше не нужно писать отдельный JavaScript-код для захвата! Движок берёт все ваши template:, автоматически компилирует их в генератор кода и создает умные регулярные выражения для обратного парсинга. Всё просто, как и весь движок!

Дополнительные примеры

Блок с вложенными блоками (input)

#\ row: "Установить переменную" (var_name: text: "x") "равной"
#\ input: value (type: value) "значение"
#\ template: val {var_name} = {value}

Блок условия (if/else)

#\ block: if_statement
#\ category: Логика
#\ color: #10b981
#\ shape: statement
#\ 
#\ row: "Если" input: condition (type: value)
#\ row: "то"
#\ input: then_do (type: statement)
#\ row: "иначе"
#\ input: else_do (type: statement)
#\ 
#\ template: if ({condition}) {{ "\n" }} {then_do} {{ "\n" }} else {{ "\n" }} {else_do}

Блок цикла (while)

#\ block: while_loop
#\ category: Циклы
#\ color: #f59e0b
#\ shape: statement
#\ 
#\ row: "Пока" input: condition (type: value)
#\ row: "выполнять"
#\ input: body (type: statement)
#\ 
#\ template: while ({condition}) {{ "\n" }} {body}

Блок с числовым полем и слайдером

#\ row: "Подождать" (seconds: number: 1) "секунд"
#\ template: await time({seconds})

Совместимость со старым форматом

Старый формат с секциями [UI], [DYNAMIC_UI], [CODE_GEN] и [CODE_PARSE] с сырым JS всё ещё поддерживается для обратной совместимости и особо сложных кастомных блоков, но использовать его не рекомендуется. Используйте новый единый синтаксис #\!

Старый формат (не рекомендуется):

[UI]
row: [npc: dropdown_npc] "сказать" (msg: text: "Привет!")

[CODE_GEN]
return `${npc}.say("${msg}")`;

Шпаргалка для быстрого старта

#\ block: my_block

Уникальный ID блока

#\ category: Моя категория

Категория в панели блоков

#\ color: #38bdf8

Цвет блока (HEX)

#\ shape: statement

Форма блока (statement/value)

#\ block: example_block
#\ category: Примеры
#\ color: #d1e65e
#\ shape: statement
#\ 
#\ row: [npc: dropdown_npc] "выполнить действие" [action: dropdown(привет: hello, пока: wave)]
#\ 
#\ if action == "hello":
#\   row: "с силой" (power: number: 1)
#\   template: {npc}.hello({power})
#\ 
#\ if action == "wave":
#\   template: {npc}.wave()
